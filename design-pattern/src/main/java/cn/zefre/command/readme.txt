命令模式：把请求封装成对象，将请求发出者与请求执行者解耦

角色：
Command:封装请求，通常是一个接口，一般来说包含execute()和undo()方法
ConcreteCommand:实际的Command，持有Receiver对象，通过调用Receiver的功能来完成命令
Invoker:请求发出者，要求Command执行操作，即调用Command的execute()方法
Receiver:请求接收者，实际执行请求的角色，由Command调用的execute()方法调用Receiver

可撤销的操作：可撤销操作的意思就是：放弃该操作，回到未执行该操作前的状态。
有两种方式：
① 一种是补偿式，又称反操作式
比如被撤销的操作是加的功能， 那撤消的实现就变成减的功能;同理被撤销的操作是打开的功能，那么撤销的实现就变成关闭的功能。
② 另外一种方式是存储恢复式
意思就是把操作前的状态记录下来，然后要撤销操作的时候就直接恢复回去就可以了。
